# Project Ideas

This file collects small, practical ideas that can grow BiS into a stronger open-source project.

## Beginner Issues

These are good first contribution candidates:

* Add one short biology or neuroscience example note
* Add tests for CLI argument handling
* Improve error messages when a note file is missing
* Add glossary entries for common bioinformatics terms
* Add a short explanation of `.csv`, `.tsv`, and `.h5ad` data files

## Documentation Projects

Useful documentation projects include:

* A beginner glossary for bioinformatics
* A reading guide for single-cell RNA-seq papers
* A short tutorial on reproducible project structure
* A guide to turning study notes into code examples
* A comparison of notebooks, scripts, and packages for students

## Code Projects

Possible small code projects:

* Better sentence splitting for Markdown notes
* Keyword extraction with custom keyword files
* A `bis-notes glossary` command
* A simple note indexer for a folder of Markdown files
* Export summaries to JSON for later analysis

## Biomedical Learning Tracks

Future learning tracks:

* Single-cell RNA-seq starter notes
* Fibrosis and immune response case study
* Organoid research reading map
* Clinical prediction basics for non-clinical students
* Biomedical AI workflow planning

## External Contribution Targets

To grow as an open-source contributor, the maintainer can look for documentation or beginner issues in:

* Biopython
* Scanpy
* scikit-bio
* napari
* MNE-Python
* pandas
* scikit-learn
* Matplotlib

Documentation, examples, tests, and issue triage are all valuable contributions.
