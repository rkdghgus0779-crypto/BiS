# Research Focus

BiS is an educational open-source project, not a clinical tool. Its research direction is to help students learn how biomedical questions can be translated into data-driven workflows.

## Central Theme

The central theme is:

> Understanding biomedical problems through data, code, and reproducible scientific explanation.

This connects biology, neuroscience, bioinformatics, and biomedical AI without requiring beginners to start from large, complex research codebases.

## Current Focus Areas

### Bioinformatics and Scientific Python

BiS starts with small Python tools because many biology students need a gentle path into programming before using larger libraries such as NumPy, pandas, Scanpy, scikit-learn, or PyTorch.

### Single-cell Thinking

Single-cell RNA-seq is a useful learning target because it connects cell types, gene expression, immune response, and disease mechanisms. Future BiS materials may include beginner-friendly explanations of:

* AnnData and `.h5ad` files
* Cell type annotation
* Clustering and visualization
* Batch correction concepts
* Reading papers that use single-cell data

### Fibrosis and Immune Response

Fibrosis is a strong example of how cell-level biology becomes a real medical problem. BiS may use fibrosis as a recurring case study to explain:

* Macrophages and fibroblasts
* Myofibroblast activation
* Extracellular matrix remodeling
* TGF-beta and related signaling
* How computational analysis can support biological interpretation

### Biomedical AI

BiS treats AI as a support tool for scientific learning, documentation, analysis planning, and reproducible workflows. The project does not present AI output as medical advice.

## What This Project Is Not

BiS is not:

* A diagnostic system
* A replacement for formal biomedical training
* A production clinical application
* A source of medical advice

It is a learning project for students who want to connect biological questions with code.
