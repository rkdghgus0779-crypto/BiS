# Open Source Growth Plan

This document tracks how BiS can grow from an early-stage repository into a more credible open-source project.

## Positioning

BiS should be positioned as:

> A beginner-friendly biomedical AI and bioinformatics learning toolkit for students who want to connect biological concepts with Python and reproducible analysis.

This is clearer than presenting it as a general study-note repository.

## Near-term Milestones

### Version 0.1

* Package structure
* CLI
* Tests
* GitHub Actions CI
* Contributor documentation
* Project direction docs

### Version 0.2

* Better Markdown note parsing
* JSON output support
* More example notes
* First glossary entries
* First tagged GitHub release

### Version 0.3

* Public dataset tutorial
* Single-cell RNA-seq reading guide
* Fibrosis and immune response case study
* Documentation site

## Metrics to Track

Track these in release notes or `docs/impact.md`:

* Stars
* Forks
* Contributors
* Merged pull requests
* Issues opened and closed
* Releases
* Package downloads if published
* External projects contributed to by the maintainer

## Claude for Open Source Readiness

The strongest path toward a future Claude for Open Source application is not visual polish alone. It is evidence of public contribution.

Important evidence to build:

* Merged pull requests in repositories not owned by the maintainer
* Clear releases and changelog entries
* A useful README with installation and examples
* CI passing on pull requests
* External contributors or users
* Public examples that solve a real learning problem

## Weekly Maintenance Rhythm

A simple weekly rhythm:

* 1 small BiS improvement
* 1 external open-source contribution attempt
* 1 documentation or example update
* 1 issue created or cleaned up
* 1 short progress note in the changelog or docs
